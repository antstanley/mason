## Anti-Patterns Verdict

**Not AI slop, but the landing page is the predictable version of itself, and the base canvas is ironic.**

**LLM assessment (A):** The wall has earned specificity: three genuinely different card anatomies, kind-coded 2px borders, the hard 0-2px-0 "mortar ledge" shadow, overshoot brick-in easing, decisions, not defaults. Three tells pull against it: (1) `--color-plaster` is the 2024–25 cream default (L 0.97, C 0.015, hue 85), literally on mason's own anti-reference list; (2) the landing is the stock "playful indie app" kit (centered wordmark, pill input, pink pill CTA, tilted emoji tiles), any model would generate approximately this; (3) six emoji carrying brand weight that type/color should carry. One SaaS-eyebrow instance (BlogCard publication chip, uppercase+tracked at 11px).

**Deterministic scan (B):** exit 2, exactly **1 finding**, `bounce-easing` at `web/src/app.css:29` (`cubic-bezier(0.22, 1.2, 0.36, 1)`, y2=1.2 overshoot on brick-in). Synthesis ruling: this is the product's signature motion (bricks settling into mortar) and A independently called it the best micro-moment, **intentional deviation, keep, but it's the exception that proves the no-bounce rule, don't spread it.** No other detector findings: no gradient text, no glassmorphism, no side-stripes, clean.

**Visual overlays:** not presentable this run (headless browser, no attended tab), evidence collected by script; zero console errors on all four page/theme combos.

## Overall Impression

The wall itself is real design, tactile, coherent, restrained where it counts. What surrounds the wall hasn't had the same love: arrival (generic landing), failure (dead ends and silent stalls), and access (focus, contrast, reduced-motion) are all a tier below the core. The biggest single opportunity: make the front door show the product, the landing hides the only thing that sells mason.

## What's Working

1. **Brick identity is structural, not painted on**, border coding + mortar-ledge shadow + radius + overshoot easing reinforce each other; it's exactly not-Pinterest, not-SaaS.
2. **URL-as-state done right**, ?actor= is the single source of truth; back/forward, titles, and the share loop all work with zero extra UI.
3. **Video policy enforced by architecture**, the <video> element doesn't exist until click; single-player claim/release prevents cacophony.

## Priority Issues

**[P0] The error page is a dead end**, FeedGrid renders heading + "try again" copy with no input, no retry, and the typo'd handle discarded. This is the most common failure of the primary flow (Jordan typing an Instagram username hits exactly this). *Fix:* inline (prefilled, focused) handle form on the error state. → `/impeccable harden`

**[P1] Endless scroll dies silently**, loadMore failure mid-scroll sets `feed.error` but FeedGrid only shows errors when items.length === 0; nothing on screen distinguishes "network died" from "no more content." *Fix:* inline "couldn't lay more bricks, tap to retry" row when error && items.length > 0. → `/impeccable harden`

**[P1] Five AA contrast failures in light mode (measured, not guessed)**, pink CTA text 3.26:1, blog publication chip 2.20:1, @handle metadata 3.88:1, landing help text 2.98:1, input placeholder 2.33:1. Dark mode passes everywhere sampled. WCAG 2.2 AA is stated product policy, this is a compliance gap. *Fix:* darkened text-variant tokens (pop-pink-deep fill or ink-on-pink, brick-blog-text at L≈0.5) and floor muted text at ~70% opacity. → `/impeccable polish` (token pass)

**[P1] Keyboard/focus debt**, zero authored :focus-visible styles anywhere (B: input is outline-none with border-color-only signal; everything else UA default); no skip link; masonry tab order is column-major (scrambled vs visual order); landing input has no label/aria-label; hearts/repost counts are bare glyphs to screen readers; video bricks have no link out at all (a cul-de-sac for every user, worst by keyboard). *Fix:* shared focus ring token, aria-labels, skip link, VideoCard href. → `/impeccable audit` then `polish`

**[P2] No reduced-motion path + re-animation bug + breakpoint mismatch**, B confirmed all 24 brick-in animations and hover transitions run under prefers-reduced-motion: reduce (there is no @media rule in the codebase); A found every column-count change replays the entire wall's entrance animation (device rotation = full re-cascade), and skeleton grid breaks at viewport 1440 while masonry measures container width (~1470 effective) so 4-col skeletons resolve into a 3-col wall. *Fix:* reduce → fade-only + no pulse/hover-transform; animate only newly appended bricks; single source for column breakpoints. → `/impeccable animate` (motion pass covers all three)

**[P2] Mobile header tap targets**, logo (106×32) and switch (72×32) both under 44px height. → fold into `polish`

## Minor Observations

- Wordmark 🧱 wrapped in text-pop-pink, emoji ignore color; the class does nothing.
- Demo wall repeats identical post texts at different like counts within 3 viewports, undermines the serendipity pitch in the most-shared first impression.
- Dark "kiln" mode chroma (0.01–0.012) is imperceptible, reads generic near-black, not fired clay; try L 0.20 / C 0.03 hue 50.
- Landing tagline orphans "follow" at 1440; min-h-[70vh] leaves the bottom half of tall viewports empty.
- Wall page has no h1; blog titles are h2s without a heading root.
- No footer/about/source link, one caption line is the entire trust surface.
- Skeleton tints previewing the kind mix is a lovely touch, keep.
- Perf is healthy: 819ms DCL on the wall, ~388KB gzipped wasm served from the SW, zero horizontal overflow at any width, all images reserve space.

## Questions to Consider

1. If the wall is the star, why does the front door hide it? A live demo wall *as* the landing background would explain the product in zero words.
2. Where does serendipity actually come from? Today the wall is strictly your own follows re-rendered, a prettier timeline. Does mason need one "stranger's brick" per screen to earn its success metric?
3. Is a playing video still a brick? It currently can't be dismissed, keeps playing off-screen, and links nowhere, neither modal nor brick.
